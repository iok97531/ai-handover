# 비즈니스 모델 검토

## 수익 모델 옵션

### 1. 직접 판매 (라이선스)
- **장점:** 초기 수익 빠름, 개인/소규모 팀에게 어필
- **단점:** 반복 수익 없음, 업데이트 동기 약해짐
- **적합한 경우:** 타겟이 대기업 1회성 구매 (SI, 공공기관)

---

### 2. 구독형 (SaaS) — 하이브리드 모델 추천

```
Free
 └─ 로컬 앱, 본인 OpenAI/Claude API 키, 프로젝트 1개

Pro (월 $15~20)
 └─ 무제한 프로젝트
 └─ 인수인계 문서 공유 링크
 └─ 팀 히스토리 공유

Enterprise (협의)
 └─ 완전 오프라인 — 코드/데이터 외부 전송 없음
 └─ 사내 LLM 서버 연동 (Ollama, vLLM, Azure OpenAI Private 등)
 └─ 온프레미스 설치
 └─ 보안 감사 대응 가능
```

> 구독의 가치를 만들려면 **팀 협업 기능**이 핵심.
> 지금 구조가 로컬 앱이라 클라우드 요소 추가가 필요함.

---

### Enterprise 전략 상세

**핵심 세일즈 포인트:**
> "소스코드가 회사 밖으로 한 줄도 나가지 않는 AI 인수인계 툴"

**타겟:**
- 금융/의료/공공: 규제상 소스코드·내부 문서를 외부 API에 못 보내는 환경
- 대형 SI/IT 서비스사: 고객사 코드를 받아 작업하는데 외부 유출 불가
- 제조/방산: 보안 등급으로 인터넷 자체가 차단된 환경

**계약 규모:** Free/Pro와 달리 건당 수천만 원 단위 계약 가능

**기술적으로 추가할 것:**

현재 `backend/ai/provider.py`에 AI 추상화 레이어가 이미 있어 추가가 용이함.

```
├─ Ollama 프로바이더 (로컬 LLM, 가장 대중적)
├─ vLLM / LM Studio 프로바이더
├─ Azure OpenAI Private Endpoint 지원
└─ 설정 UI에서 커스텀 API 엔드포인트 입력
```

---

### 3. 가장 현실적인 단기 수익: API 중계 방식

사용자가 자기 API 키를 넣는 대신, 서비스에서 AI 호출을 대신 처리하고 사용량만큼 과금.

- 사용자는 OpenAI 가입 불필요 → 진입장벽 낮아짐
- AI 호출 비용 + 마진을 구독료에 포함
- 사용량 기반 과금도 가능

## 타겟 시장

| 타겟 | 플랜 | 이유 |
|------|------|------|
| **프리랜서 개발자** | Free/Pro | 여러 프로젝트 관리, 고객 인계 필요 |
| **개발팀 있는 스타트업** | Pro | 온보딩 비용이 크고 문서화가 부족 |
| **IT 아웃소싱 회사** | Pro/Enterprise | 프로젝트 인수인계가 반복적으로 발생 |
| **금융/의료/공공기관** | Enterprise | 외부 API 사용 규제, 완전 오프라인 필요 |
| **제조/방산** | Enterprise | 인터넷 차단 환경, 사내 LLM 운영 중 |

## 라이선스 체크 전략

### 플랜별 방식

| 플랜 | 방식 | 이유 |
|------|------|------|
| Free | 없음 | 그냥 사용 |
| Pro | 온라인 구독 체크 (30일 캐시) | 자동 갱신, 관리 편함 |
| Enterprise | 연간 오프라인 라이선스 키 | 보안 환경 대응, 대기업 구매팀 친숙 |

### Pro: 온라인 구독 체크
```
앱 실행 → 라이선스 서버 체크 → 유효하면 동작
 └─ 30일 캐시: 인터넷 끊겨도 30일은 동작
 └─ 구독 해지 시 30일 후 자동 차단
```

### Enterprise: 오프라인 라이선스 키
```
계약 체결 → 머신 ID 수집 → 키 발급 → 앱에 입력
 └─ 키 안에 만료일 포함 (서명 검증으로 위변조 방지)
 └─ 인터넷 없이 로컬에서만 검증
```

**갱신 프로세스:**
```
만료 30일 전 앱에서 경고 표시
 └─ 담당자가 갱신 요청 (이메일/포털)
 └─ 결제 확인 후 새 키 발급 (이메일 전달)
 └─ 앱 설정에서 새 키 입력 → 즉시 적용
 ※ 기존 데이터/설정은 그대로 유지
```

**키 구조 (JWT 방식):**
```
{
  "company": "삼성SDS",
  "seats": 50,
  "issued_at": "2025-01-01",
  "expires_at": "2026-01-01",
  "machine_ids": ["abc123", "def456", ...]  // 등록된 기기 목록
}
→ 비대칭 키(RSA/ECDSA)로 서명 → 앱에 공개키 내장해서 검증
```

**머신 ID 초과 시 (기기 교체 등):**
- 포털 또는 이메일로 기기 해제 요청
- 담당자가 확인 후 새 키 재발급

### 구현 시 재사용 가능한 기존 코드
- `backend/services/crypto.py` — 암호화/복호화 로직 재사용 가능
- `cryptography` 라이브러리 이미 설치됨

## 권장 전략

**무료 로컬 버전으로 사용자를 모으고 → 팀 협업 기능으로 구독 전환 → Enterprise로 확장**

1. 무료 버전 배포 (로컬, 본인 API 키)
2. 팀 협업 기능 개발 (공유, 클라우드 저장)
3. Pro 구독 도입 (온라인 체크)
4. Enterprise 라이선스 도입 (오프라인 키, 사내 LLM 연동)
5. 장기적으로 API 중계 방식으로 진입장벽 낮추기
