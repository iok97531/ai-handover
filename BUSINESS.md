# 비즈니스 모델 검토

> **비전:** 프로젝트의 지식이 사람이 아니라 시스템에 축적되는 세상 ([VISION.md](VISION.md))
> **포지셔닝:** 프로젝트의 모든 파일을 읽고 이해하는 AI — 코드와 문서를 동시에
> **핵심 문제:** 프로젝트 지식이 사람 머리에만 있어서, 사람이 떠나면 지식도 사라진다. Scope는 이미 존재하는 파일에서 지식을 추출하여 누구든 접근 가능하게 만든다.

---

## 왜 돈을 내는가

1. **시간 절약** — 수백 개 파일을 직접 읽는 대신 AI에게 물어본다 (개인 가치)
2. **지식 보존** — 팀원이 떠나도 프로젝트 지식이 유지된다 (조직 가치)
3. **보안** — 코드와 문서가 외부로 나가지 않는다 (Enterprise 가치)

---

## 수익 모델 — 구독형 (하이브리드)

```
Free — 개인 체험
 └─ 로컬 앱, 본인 API 키, 프로젝트 1개
 └─ 프리랜서, 학습 목적

Starter (월 ₩15,000 / $12) — 개인 실무
 └─ 문서 100개, 월 500회 질문
 └─ 기본 템플릿, PDF 내보내기
 └─ 1인 사용 (프리랜서, 1인 개발자)

Team (월 ₩25,000/인 / $20/인) — 소기업·스타트업 (5~30명)
 └─ 무제한 문서·질문
 └─ 팀 공유, 공유 링크
 └─ 업종별 템플릿
 └─ 인계자 브레인덤프 (퇴사/이관자의 암묵지를 AI가 구조화하여 인덱싱)
 └─ 감사 로그

Enterprise (협의) — 중견·대기업, 규제 산업
 └─ 완전 오프라인 — 코드/문서/데이터 외부 전송 없음
 └─ 사내 LLM 서버 연동 (Ollama, vLLM, Azure OpenAI Private 등)
 └─ 온프레미스 설치
 └─ SSO, RBAC, 감사 로그
 └─ 보안 감사 대응 가능
```

> 구독의 가치를 만들려면 **팀 내 인수인계 기능**이 핵심. 인계자 브레인덤프는 "퇴사 프로세스에 Scope를 포함"하게 만드는 제도적 lock-in 기능.
> 지금 구조가 로컬 앱이라 클라우드 요소 추가가 필요함.

### Enterprise 전략 상세

**핵심 세일즈 포인트:**
> "소스코드도 프로젝트 문서도 회사 밖으로 한 줄도 나가지 않는 AI 프로젝트 지식 관리 툴"

**타겟:**
- 금융/의료/공공: 규제상 소스코드·내부 문서를 외부 API에 못 보내는 환경
- 대형 SI/IT 서비스사: 고객사 코드·문서를 받아 작업하는데 외부 유출 불가
- 제조/방산: 보안 등급으로 인터넷 자체가 차단된 환경
- 법무법인/컨설팅: 고객 기밀 문서 보안, 대량 문서 분석 수요

**계약 규모:** Free/Starter와 달리 건당 수천만 원 단위 계약 가능

**기술적으로 추가할 것:**

현재 `backend/ai/provider.py`에 AI 추상화 레이어가 이미 있어 추가가 용이함.

```
├─ Ollama 프로바이더 (로컬 LLM, 가장 대중적)
├─ vLLM / LM Studio 프로바이더
├─ Azure OpenAI Private Endpoint 지원
└─ 설정 UI에서 커스텀 API 엔드포인트 입력
```

---

### API 중계 방식 — Phase 5에서 도입

사용자가 자기 API 키를 넣는 대신, 서비스에서 AI 호출을 대신 처리하고 사용량만큼 과금.

- 사용자는 OpenAI 가입 불필요 → 비개발자 진입장벽 제거
- AI 호출 비용 + 마진을 구독료에 포함
- 사용량 기반 과금도 가능
- Free 플랜은 본인 API 키 유지 (비용 통제)

> **로컬 우선 원칙과의 관계:** API 중계 시 질문과 관련된 텍스트 조각이 Scope 서버를 경유한다. 이는 사용자가 **명시적으로 선택**한 경우에만 활성화되며, 기본값은 여전히 본인 API 키 직접 사용이다. Enterprise 플랜은 API 중계를 사용하지 않는다 (완전 오프라인).

**AI 비용 관리 전략:**
- 모델 선택: Free도 충분한 품질의 모델을 사용 — 첫 경험이 나쁘면 전환이 안 됨 (점진적 가치 원칙)
- 캐싱: 동일/유사 질문 결과 캐싱으로 중복 호출 절감
- 청크 최적화: 불필요한 컨텍스트를 줄여 토큰 사용량 절감
- 사용량 한도: Free 플랜에 월간 질문 횟수 제한으로 비용 통제

## 경쟁 전략

### 왜 경쟁자가 아니라 Scope를 쓰는가

| 경쟁자 | 강점 | Scope와의 차이 |
|--------|------|---------------|
| **GitHub Copilot / Cursor** | 코드 작성·편집 최강 | 코드만 봄. 문서를 모름. Scope는 읽기 전용이지만 코드+문서를 모두 이해 |
| **ChatPDF / ChatDOC** | PDF Q&A 간편함 | 코드를 모름. 프로젝트 단위가 아닌 파일 단위. 클라우드 전송 필수 |
| **Notion AI** | 기존 Notion 사용자에게 편리 | Notion 안의 문서만 봄. 코드·로컬 파일 접근 불가 |
| **Glean** | 기업 내부 검색 통합 | 대기업 전용 ($$$). 코드 이해 약함. 완전 클라우드 |
| **Microsoft Copilot** | M365 통합 | M365 생태계 안에서만 동작. 코드 이해 약함. 오프라인 불가 |

### Scope의 포지션

```
         코드 이해
            ↑
  Copilot   |   ★ Scope
  Cursor    |
            |
 ───────────┼──────────→ 문서 이해
            |
            |   Notion AI
            |   ChatPDF
            |   Glean
```

Scope는 코드 이해와 문서 이해가 교차하는 유일한 위치에 있다. 여기에 로컬/오프라인이라는 축이 더해지면 보안 중심 조직에서는 대안이 없다.

|------|------|------|

## 라이선스 체크 전략

### 플랜별 방식

| 플랜 | 방식 | 이유 |
|------|------|------|
| Free | 없음 | 그냥 사용 |
| Starter/Team | 온라인 구독 체크 (30일 캐시) | 자동 갱신, 관리 편함 |
| Enterprise | 연간 오프라인 라이선스 키 | 보안 환경 대응, 대기업 구매팀 친숙 |

### Starter/Team: 온라인 구독 체크
```
앱 실행 → 라이선스 서버 체크 → 유효하면 동작
 └─ 30일 캐시: 인터넷 끊겨도 30일은 동작
 └─ 구독 해지 시 30일 후 자동 차단
```

### Enterprise: 오프라인 라이선스 키
```
계약 체결 → 머신 ID 수집 → 키 발급 → 앱에 입력
 └─ 키 안에 만료일 포함 (서명 검증으로 위변조 방지)
 └─ 인터넷 없이 로컬에서만 검증
```

**갱신 프로세스:**
```
만료 30일 전 앱에서 경고 표시
 └─ 담당자가 갱신 요청 (이메일/포털)
 └─ 결제 확인 후 새 키 발급 (이메일 전달)
 └─ 앱 설정에서 새 키 입력 → 즉시 적용
 ※ 기존 데이터/설정은 그대로 유지
```

**키 구조 (JWT 방식):**
```
{
  "company": "삼성SDS",
  "seats": 50,
  "issued_at": "2025-01-01",
  "expires_at": "2026-01-01",
  "machine_ids": ["abc123", "def456", ...]  // 등록된 기기 목록
}
→ 비대칭 키(RSA/ECDSA)로 서명 → 앱에 공개키 내장해서 검증
```

**머신 ID 초과 시 (기기 교체 등):**
- 포털 또는 이메일로 기기 해제 요청
- 담당자가 확인 후 새 키 재발급

### 구현 시 재사용 가능한 기존 코드
- `backend/services/crypto.py` — 암호화/복호화 로직 재사용 가능
- `cryptography` 라이브러리 이미 설치됨

## 권장 전략

**무료 로컬 버전으로 사용자를 모으고 → 문서 확장으로 시장을 넓히고 → 팀 협업 기능으로 구독 전환 → Enterprise로 확장**

> 전 과정에서 [제품 원칙](VISION.md#제품-원칙)을 유지한다: 로컬 우선(클라우드는 선택), 제로 노력(폴더 지정만으로 동작), 보안 절대주의(기본값이 안전).

1. 무료 버전 배포 (로컬, 본인 API 키, 코드 인수인계)
2. 문서 포맷 확장 (PDF, DOCX, PPTX — 비개발자도 사용 가능)
3. 문서 중심 UX 추가 (분석 템플릿, 용어 이중화)
4. 검색 고도화 + AI 답변 내보내기 (하이브리드 검색, 자동 태그)
5. 업종 특화 + Starter/Team 구독 도입 (API 중계 옵트인, Stripe 연동)
6. Enterprise 라이선스 도입 (오프라인 키, 사내 LLM 연동, SSO/RBAC)

상세 일정은 [ROADMAP.md](ROADMAP.md) 참조.
