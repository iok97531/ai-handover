# Scope — 제품 로드맵

> **비전:** 프로젝트의 지식이 사람이 아니라 시스템에 축적되는 세상 ([VISION.md](VISION.md))
> **포지셔닝:** 프로젝트의 모든 파일을 읽고 이해하는 AI — 코드와 문서를 동시에
> **전략:** Free → 문서 확장 → Pro/Team → Enterprise 순서로 단계적 확장
> 각 Phase는 이전 Phase의 사용자 기반과 피드백을 기반으로 진행한다.
> 모든 결정은 [제품 원칙](VISION.md#제품-원칙)에 따른다: 로컬 우선, 제로 노력, 보안 절대주의, 점진적 가치, 개발자+비개발자 모두.

---

## Phase 1 — Free 출시 (개발자 코드 인수인계)
> **목표:** 사용자 확보 및 피드백 수집

- [x] 핵심 기능 완성 (코드 인덱싱, AI Q&A, 인수인계 4종)
- [x] 랜딩페이지 제작 (getscope.dev)
- [ ] PyInstaller 번들링 — Python 설치 없이 바로 실행 가능
- [ ] Windows 배포 (exe 설치 파일)
- [ ] Free 플랜 공개 배포 (프로젝트 1개, 본인 API 키 사용)

**점진적 가치 검증:**
- [ ] 폴더 등록 → 인덱싱 → 첫 질문 답변까지 5분 이내 확인
- [ ] API 키 설정을 포함한 최초 실행 → 가치 체감까지 10분 이내 확인

---

## Phase 2 — 문서 포맷 확장 (4~6주)
> **목표:** 코드뿐 아니라 비코드 문서도 인덱싱할 수 있게 확장
> 기존 개발자 기능은 그대로 유지한다.

**백엔드:**
- [ ] PDF 파서 추가 (`PyMuPDF`)
- [ ] DOCX 파서 추가 (`python-docx`)
- [ ] PPTX 파서 추가 (`python-pptx`)
- [ ] XLSX/CSV 파서 추가 (`openpyxl` + `pandas`)
- [ ] `config.py` 지원 확장자 업데이트
- [ ] `MAX_FILE_SIZE` 50MB로 확장
- [ ] 문서 메타데이터 추출 (작성자, 날짜, 페이지수)

**프론트엔드:**
- [ ] 문서 유형별 아이콘 표시
- [ ] 인덱싱 진행률에 문서 유형 정보 추가

**검증:** 코드 프로젝트 + PDF/DOCX가 혼합된 폴더에서 정상 인덱싱 및 Q&A 확인

---

## Phase 3 — 문서 중심 UX (4~6주)
> **목표:** 비개발자도 이해할 수 있는 UI 레이어 추가

**프론트엔드:**
- [ ] 폴더 추가를 보조하는 드래그앤드롭 (핵심은 여전히 "폴더 지정 → 자동 인덱싱")
- [ ] 문서 목록 뷰 (유형, 날짜, 크기, 상태)
- [ ] 범용 분석 템플릿 추가 (기존 인수인계 5종 유지 + 프로젝트 현황 요약, 핵심 문서 요약, Q&A 보고서, 비교 분석 등 4~6종 추가)
- [ ] 용어 이중화: 설정에서 "개발자 모드 / 일반 모드" 선택

**백엔드:**
- [ ] 시맨틱 청킹 옵션 추가 (문서 유형 감지 → 자동 전략 선택)
- [ ] 인용 형식 변경: 문서명 + 페이지번호 (코드는 기존 파일:라인 유지)
- [ ] 시스템 프롬프트를 프로젝트 유형에 따라 분기

**검증:** 비개발자 테스터 3~5명에게 문서 등록 → Q&A → 유용성 피드백 수집

---

## Phase 4 — 검색 고도화 + 내보내기 (4~6주)
> **목표:** 검색 품질과 결과물 활용성 강화

**백엔드:**
- [ ] BM25 키워드 검색 추가 (하이브리드 검색)
- [ ] 메타데이터 필터 (날짜 범위, 문서 유형)
- [ ] AI 자동 태그 분류 — 문서 내용 기반으로 태그를 자동 생성 (수동 태그 아님, 제로 노력 원칙)
- [ ] AI 답변 내보내기 API (마크다운 → PDF/DOCX) — 답변 공유 목적, 문서 편집 기능은 넣지 않음

**프론트엔드:**
- [ ] 검색 필터 UI (날짜, 유형, 자동 생성 태그)
- [ ] 내보내기 버튼 (AI 답변을 PDF/DOCX로 저장)

**검증:** 50개+ 문서가 등록된 프로젝트에서 다양한 질문 유형의 정확도 측정

---

## Phase 5 — 업종 특화 + Pro/Team 출시 (6~8주)
> **목표:** 첫 유료 수익, 특정 업종에서 차별화

**플랜 체계:**
```
Free
 └─ 로컬 앱, 본인 API 키, 프로젝트 1개
 └─ 개인 사용 (프리랜서, 학습 목적)

Starter (월 ₩15,000 / $12)
 └─ 문서 100개, 월 500회 질문
 └─ 기본 템플릿, PDF 내보내기
 └─ 1인 사용 (프리랜서, 1인 개발자 — 본인 프로젝트 정리)

Team (월 ₩25,000/인 / $20/인)
 └─ 무제한 문서·질문
 └─ 팀 공유, 공유 링크
 └─ 업종별 템플릿
 └─ 인계자 브레인덤프
 └─ 감사 로그
 └─ 소기업·스타트업 대상 (5~30명, 팀 내 인수인계가 발생하는 조직)
```

**기능:**
- [ ] 업종별 템플릿 패키지 (법무, 마케팅, 경영기획 중 1개 선정)
- [ ] 업종별 시스템 프롬프트 최적화
- [ ] API 중계 방식 도입 — 사용자가 명시적으로 선택한 경우에만 활성화 (로컬 우선 원칙: 기본값은 여전히 본인 API 키)
- [ ] 사용량 관리 + 과금 시스템
- [ ] 라이선스 서버 구축 (온라인 구독 체크 + 30일 로컬 캐시)
- [ ] Starter/Team 결제 연동 (Stripe)
- [ ] macOS 지원

**인계자 브레인덤프 (Team 플랜):**
> 퇴사/이관하는 사람이 두서없이 쏟아낸 지식을 AI가 구조화하여 프로젝트에 인덱싱한다. 인수자는 코드/문서와 함께 이 맥락을 검색할 수 있다.
- [ ] 브레인덤프 채팅 UI — 인계자가 자유롭게 입력하면 AI가 실시간 구조화 (아키텍처 결정, 운영 주의사항, 알려진 이슈, 주요 연락처 등)
- [ ] 구조화된 결과를 프로젝트에 연결 → 청킹 → 임베딩 → 기존 RAG 파이프라인에 통합
- [ ] 인수자가 Q&A 시 코드/문서/브레인덤프 맥락이 함께 검색됨

**클라우드 연동 (Team 플랜, 옵트인 방식):**
> 로컬 우선 원칙 유지: 클라우드에서 가져온 문서는 로컬에 저장 후 인덱싱. 원본은 클라우드에 남고, Scope는 사본만 보유.
- [ ] Google Drive 연동 — OAuth 인증 → gDocs를 DOCX로, gSheets를 XLSX로 자동 export → 로컬 저장 → 인덱싱
- [ ] Google Drive 변경 감지 — 문서 수정 시 자동 재인덱싱 (polling 또는 Drive Push Notifications)
- [ ] Notion 연동 — API로 페이지 export → 로컬 저장 → 인덱싱
- [ ] 연동 설정 UI — 클라우드 소스 추가/제거, 동기화 상태 표시

**검증:** 타겟 업종 실제 기업 2~3곳에서 파일럿 테스트

## Phase 6 — Enterprise 출시 (8~12주)
> **목표:** 고단가 B2B 계약 (건당 수천만 원)
> **세일즈 포인트:** "소스코드도 문서도 회사 밖으로 한 줄도 나가지 않는다"

**보안/협업:**
- [ ] 사용자 인증 (로그인/회원가입, SSO)
- [ ] 프로젝트별 접근 권한 (RBAC)
- [ ] 감사 로그 (누가 언제 어떤 문서에 질문했는지 기록)
- [ ] 팀 공유 기능

**문서 확장:**
- [ ] HWP 파싱 (한국 시장 공략)
- [ ] OCR (스캔 문서, 이미지 텍스트 추출)

**오프라인/온프레미스:**
- [ ] 로컬 LLM 프로바이더 연동 (Ollama, vLLM, LM Studio)
- [ ] Azure OpenAI Private Endpoint 지원
- [ ] 설정 UI에서 커스텀 API 엔드포인트 입력
- [ ] 완전 오프라인 모드 (로컬 LLM + 로컬 임베딩)
- [ ] 온프레미스 설치 패키지

**라이선스:**
- [ ] 오프라인 라이선스 키 시스템 (JWT + RSA/ECDSA 서명)
- [ ] 머신 ID 기반 발급, 만료일 포함
- [ ] 갱신: 만료 30일 전 경고 → 이메일로 새 키 발급
- [ ] 보안 감사 대응 문서

**요금:** 시트 라이선스 (카피당 x 연간 계약), 협의

**주요 타겟:**
| 타겟 | 이유 |
|------|------|
| 금융·의료·공공기관 | 외부 API 사용 규제, 데이터 주권 요건 |
| 대형 SI·IT 서비스사 | 고객사 코드·문서 외부 유출 금지 계약 |
| 제조·방산 | 인터넷 차단 환경, 사내 LLM 운영 중 |
| 법무법인·컨설팅 | 고객 기밀 문서 보안, 대량 문서 분석 수요 |

## 전체 타임라인 요약

```
[현재]  개발자 코드 인수인계 도구
   ↓ Phase 1 (Free 출시)
[+0주]  무료 배포, 사용자 확보
   ↓ Phase 2 (문서 포맷 확장)
[+4주]  코드 + PDF/DOCX/PPTX를 모두 인덱싱하는 프로젝트 AI
   ↓ Phase 3 (문서 중심 UX)
[+8주]  개발자/비개발자 모두 사용할 수 있는 프로젝트 AI
   ↓ Phase 4 (검색 + 내보내기)
[+12주] 실무에서 쓸 수 있는 문서 분석 도구
   ↓ Phase 5 (업종 특화 + Pro/Team)
[+18주] 첫 유료 수익, 특정 업종에서 차별화
   ↓ Phase 6 (Enterprise)
[+26주] 보안이 중요한 조직의 프로젝트 지식 관리 플랫폼
```
